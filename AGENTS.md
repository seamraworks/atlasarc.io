# AtlasArc Website Repo Instructions

## Parent-First Inspection

When this repo is checked out inside the Seamra Works workspace, start in the
parent governance repo before inspecting local website source:

1. Read `../AGENTS.md`.
2. Read `../repos.yml`.
3. Read relevant governance docs under `../docs/`, especially
   `../docs/PUBLICATION_BOUNDARY.md`, `../docs/CONTENT_GOVERNANCE.md`, and
   `../docs/website/`.
4. Return here, then inspect `README.md`, `manifest/`, and `src/`.

If the parent governance repo is unavailable, this file and `README.md` are the
local fallback.

## Boundary

This public repo contains publishable AtlasArc.io website source only.

- Do not add private content plans, launch notes, competitive notes, pricing
  rationale, or standalone design/styleguide explorations here.
- Private website planning lives in the parent governance repo at
  `../docs/website/`.
- Standalone styleguide variants live in the parent governance repo at
  `../design/styleguide-variants/`.

## Brand

Use AtlasArc for the product, AtlasArc.io for the website/product presence, and
Seamra Works for publisher/vendor/legal-owner language. Whole-product language
should mention Java, Kotlin, and TypeScript; implementation-specific language may
name only the relevant backend.
